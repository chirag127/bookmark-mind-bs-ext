## Detailed description (max 16,000 chars — Markdown-lite)

```
BookmarkMind uses AI to automatically categorize your Chrome bookmarks into a
clean, functional folder hierarchy. Instead of nesting bookmarks by who
provides them, it groups them by what they DO — so all your VPN services
land in one folder, all your code repositories in another, all your recipe
sites together.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
KEY FEATURES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

★ 13 built-in providers — bring your own key for any OpenAI-compatible LLM:
   • Groq (permanent free tier, 30 RPM)
   • Cerebras (1M tokens/day free, no card)
   • Google Gemini (free tier via OpenAI-compat layer)
   • OpenRouter (aggregator with 20+ :free models)
   • Mistral, HuggingFace Router, Novita, DeepSeek, OpenAI
   • LM Studio & Ollama (localhost, run models locally)
   • LiteLLM Proxy (self-host and route anywhere)
   • OmniRoute (localhost, 60+ free models via local dev server)

★ Custom provider — add ANY OpenAI-compatible endpoint (self-hosted vLLM,
  corporate proxy, or niche providers). Configure base URL + auth scheme.

★ Provider fallback — order providers by preference. If one hits a rate
  limit (HTTP 429), BookmarkMind cools it off for 5 minutes and tries the
  next. Auth errors stop the chain.

★ Encrypted key storage — API keys AES-GCM encrypted before writing to
  Chrome sync storage. Defense-in-depth vs another extension harvesting
  your keys.

★ FMHY-style functional categorization — hierarchical folders like
  "Tools > File Tools > Cloud Storage" rather than "Google > Drive".
  Categories are generated by the LLM based on your actual bookmarks.

★ Learning — records folder moves you make manually and feeds them back
  into future categorizations. Corrections improve over time.

★ Snapshot before reorganize — creates a full backup before rearranging so
  nothing is ever lost.

★ Rate-limit aware — per-provider request queues + retry with exponential
  backoff + rate-limit penalty tracking. Won't burn through free tiers.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
HOW IT WORKS

1. Install BookmarkMind and open the Options page.
2. Click "+ Add Provider" and pick from 13 built-in options, OR click
   "+ Custom" to add any OpenAI-compatible endpoint.
3. Paste your API key (or leave blank for localhost providers). Click Test.
4. Drag providers to reorder your fallback preference.
5. Open the BookmarkMind popup and click "Categorize All Bookmarks".
6. Watch as BookmarkMind analyzes your bookmarks, generates a functional
   folder hierarchy, and moves each bookmark to the right place.

BookmarkMind requests:
• bookmarks — to read + reorganize your bookmark tree.
• storage — to save your provider keys (encrypted) + categorization state.
• tabs — to fetch live page titles for better categorization.
• notifications — to notify you when a batch completes.
• alarms — to resume categorization if the service worker is suspended.
• Host permissions (all URLs) — needed because YOU choose which provider
  endpoints to call. We can't enumerate them all upfront. BookmarkMind
  ONLY calls the endpoints YOU configure — never any others.

Your API keys are encrypted with AES-GCM before storage. They never leave
your device except when you make an LLM call to a provider you configured.
```

## Single purpose statement (privacy tab)

```
BookmarkMind's single purpose is to automatically organize your Chrome
bookmarks into a functional folder hierarchy using AI. It reads your
bookmark tree, sends bookmark titles and URLs to an OpenAI-compatible LLM
provider that you configure, and moves each bookmark into an AI-suggested
folder based on what the site does.
```

## Permission justifications (privacy tab — MANDATORY per permission)

**`bookmarks`**
```
BookmarkMind reads the user's bookmark tree to identify uncategorized
bookmarks and rearrange them into AI-generated folders. Every operation
is initiated by an explicit user click ("Categorize All Bookmarks"). No
bookmark data is shared with anyone other than the LLM provider the user
configured.
```

**`storage`**
```
BookmarkMind stores per-provider API keys (AES-GCM encrypted) in
chrome.storage.sync so they roam across the user's Chrome installs. It
also stores user preferences (batch size, provider fallback order) and
persistent categorization state (so a large categorization can survive
service worker restarts).
```

**`tabs`**
```
BookmarkMind reads currently-open tab titles to enrich bookmark
categorization — a live tab title is more accurate than a stale bookmark
title stored years ago. It never modifies or navigates tabs, only reads
titles.
```

**`notifications`**
```
BookmarkMind shows a system notification when a categorization batch
completes so the user knows to check the result without leaving their
current tab.
```

**`alarms`**
```
BookmarkMind schedules alarms to resume batch categorization after the
Chrome service worker suspends between batches (Chrome suspends idle
service workers to save memory).
```

**`activeTab`**
```
BookmarkMind reads the currently-active tab's title when the user clicks
the extension icon, so it can offer to categorize the current page as a
bookmark.
```

**`<all_urls>` host permission (JUSTIFY EXTENSIVELY — CWS reviews this)**
```
BookmarkMind supports 13 built-in OpenAI-compatible LLM providers PLUS
a "custom provider" feature where the user enters any HTTPS endpoint.
Because the extension cannot enumerate all possible provider URLs in
advance (users may point at self-hosted vLLM, corporate proxies,
localhost:1234 for LM Studio, or any other OpenAI-compat server),
<all_urls> is required.

BookmarkMind ONLY makes network requests to endpoints the user has
explicitly configured in the Options page. It never makes background
network requests, never phones home, and has no telemetry. The user's
provider configuration in chrome.storage.sync is the ONLY source of
truth for network destinations.

Alternative (optional_host_permissions) was considered but rejected
because it would trigger a permission dialog every time the user adds a
new provider, which interrupts batch categorization flows across
multiple providers.
```
